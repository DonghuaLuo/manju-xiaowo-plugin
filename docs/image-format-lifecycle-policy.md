# Manju 图片格式生命周期方案

本文档定义 manju 插件中图片从用户输入、本地存储、前端查看、供应商传输到供应商输出落地的统一处理规则。

## 总原则

- 本策略只约束从当前流程新进入项目资产的图片、发给外部供应商的图片、以及供应商新返回并落地的图片。
- 新的用户输入图片如果要保存为项目资产，统一通过项目图片保存入口转换为真实 PNG。
- 本地已有项目资产、历史版本文件、归档导入/恢复出来的旧图片不做追溯转换；它们可以保留实际原格式。
- 前端预览直接读取项目资产原图路径，不额外生成预览压缩图，也不假设本地文件一定是 PNG。
- Agent 对话历史图片不是项目资产，不强制转 PNG，也不能因为是“临时输入”而随意删除。
- 给外部供应商传输图片时统一走传输准备入口，默认转换为高质量 JPG；若图片存在透明通道，则保持 PNG 传输。
- 供应商输出可按供应商/模型能力请求指定返回格式；收到后先检测真实图片类型，再统一转换为 PNG 存入项目资产目录。
- 全局设置 / 项目设置中的供应商输出格式只影响供应商返回偏好，不改变本地最终存储格式。

## 图片类型边界

### 项目资产图片

项目资产图片指需要保存到当前项目目录，并会被后续角色、道具、场景、分镜、视频生成等流程复用的图片。

本节规则只适用于“新进入项目资产”的图片。已有项目资产、历史版本文件、归档恢复出的旧图片不纳入强制转换范围。

包括但不限于：

- 角色集图片
- 角色设计图
- 角色参考图
- 道具库图片
- 场景库图片
- 分镜图
- 空格图
- Agent 流程生成后被写入项目资产的图片
- 用户明确添加到项目资产库的图片

新入库处理规则：

- 全部通过统一项目图片保存入口写入。
- 保存前检测真实图片类型，不能只信任文件后缀。
- 入库时统一转换为真实 PNG。
- 当前新流程中只写入最终 PNG，不额外保存原始 JPG/WebP/供应商临时文件。

不要求：

- 扫描并转换已有项目资产。
- 转换历史版本目录中的图片。
- 导入、恢复、回滚历史图片时强制转 PNG。

### Agent 对话历史图片

Agent 对话历史图片指用户在 agent 对话输入框添加的图片，或 agent 对话过程中作为消息附件保存的图片。

处理规则：

- 它们不是项目资产，不需要单独转换为 PNG。
- 它们属于对话历史，不能按普通临时文件随意删除。
- 保存策略跟随 agent 对话历史附件机制，保持原始可追溯。
- 当这些图片被选中并传给供应商时，只在传输准备阶段临时转换，不改变对话历史中的原图。

### 传输临时图片

传输临时图片指为了调用外部供应商 API 临时生成的图片。

处理规则：

- 不属于项目资产。
- 不属于 agent 对话历史。
- 可在供应商请求完成后删除。
- 只用于本次 API 调用。

## 用户输入处理

用户输入图片先判断归属：

1. 如果新的用户输入图片要进入角色、道具、场景、分镜等项目资产目录，则走项目图片保存入口，统一转 PNG 保存。
2. 如果图片只是 agent 对话输入框附件，则按 agent 对话历史附件保存，不强制转 PNG。
3. 如果图片只是一次性 API 调用过程中的中间文件，则作为传输临时图片，用完可清理。

项目图片保存入口必须完成：

- 读取图片真实内容。
- 修正 EXIF 方向。
- 转换到统一色彩空间。
- 检测是否有透明通道。
- 转换并保存为真实 PNG。
- 不额外维护图片元数据；需要宽高、MIME、文件大小等信息时直接从图片文件读取。

## 本地存储规则

从当前流程新写入的项目资产图片统一保存为 PNG。

本地项目中已经存在的图片不一定都是 PNG；历史版本、旧项目迁移、归档导入、版本回滚等路径可以保留或复制图片的实际格式，不做额外转码。

要求：

- 新输入入库后文件后缀为 `.png`。
- 新输入入库后文件真实内容必须是 PNG。
- 新输入入库路径不允许 JPEG 字节写入 `.png` 文件路径。
- 不为前端预览额外创建压缩图。
- 不为同一项目资产保留多份格式副本。
- 不为图片额外创建 sidecar 配置文件，也不把业务元数据写入图片内嵌信息。

推荐形成唯一入口：

- `save_project_image(...)`：所有项目资产图片入库入口。
- 角色、道具、场景、分镜等“新输入入库”业务代码禁止绕过该入口直接写图片文件。

版本历史规则：

- 从版本历史获取图片、恢复当前分镜或设计图时，应按历史文件的实际格式复制或恢复。
- 不把版本历史恢复视为新的用户输入入库。
- 不为了匹配 `.png` 规范而强制转码历史版本。

## 前端预览规则

前端查看图片时直接使用项目资产原图路径。

要求：

- 不生成额外 preview 文件。
- 不生成缩略图缓存。
- 不把预览图作为后续供应商输入。
- 缓存刷新问题通过更新时间、版本号或 query cache bust 处理。
- 前端不要依赖固定扩展名判断图片真实格式。

## 供应商传输规则

任何图片只要要传给外部供应商，都必须走统一传输准备入口。

包括：

- 图生图
- 图生视频
- Agent 对话中选择的图片
- Agent 自动流程中调用的图片
- 项目资产图片作为参考图传输
- 对话历史图片作为参考图传输

推荐形成统一入口：

- `prepare_provider_image(...)`

传输准备规则：

1. 先读取真实图片内容并检测是否有透明通道。
2. 如果图片有透明通道，则保持 PNG 传输，避免透明信息丢失。
3. 如果图片没有透明通道，则转换为高质量 JPG 传输。
4. JPG 质量建议使用 `92` 到 `95`；当前默认使用 `92`。
5. 可按供应商限制控制最大长边，例如 `2048` 或供应商明确上限。
6. 当请求体过大需要重试时，优先降低最大长边；JPEG 质量仍应尽量保持默认值，避免明显损伤参考图细节。
7. 传输转换不改变原项目资产，也不改变 agent 对话历史附件。

### 按传输方式处理

如果供应商接口使用 base64、data URI 或 multipart 内存数据：

- 直接在内存中从 PNG/原图转换为传输所需 JPG 或 PNG bytes。
- 不创建临时文件。

如果供应商接口必须使用文件上传：

- 将待传输图片转换成临时 JPG 或 PNG 文件。
- 上传完成后清理临时文件。
- 临时文件不得写入项目资产目录。

## 供应商输出处理

供应商输出图片必须走统一落地入口。

推荐形成统一入口：

- `save_provider_output_image(...)`

处理规则：

1. 不信任 URL、文件名、Content-Type 或扩展名，优先检测真实图片内容。
2. 如果供应商返回 JPG、WebP、PNG 或其他可解码图片格式，先解码。
3. 统一转换为真实 PNG。
4. 只把最终 PNG 写入项目资产目录。
5. 供应商原始 JPG/WebP/临时下载文件在转换成功后可删除。

注意：

- 这里的“供应商输出”只指本次供应商调用新返回、并需要落地到项目资产中的图片。
- 历史版本恢复、旧资产复制、归档导入不属于供应商输出落地入口，不在这里强制转码。

## 全局设置和项目设置

图片模型如果支持指定输出格式，需要在全局设置和项目设置中展示输出格式下拉。

示例选项：

- `png`
- `jpg`
- `webp`
- `auto`

规则：

- 该设置表示“请求供应商时希望供应商返回什么格式”。
- 如果用户选择 `jpg`，则向支持该参数的供应商请求 JPG 输出。
- 如果用户选择 `png`，则向支持该参数的供应商请求 PNG 输出。
- 如果供应商或模型不支持指定输出格式，则隐藏或禁用该下拉项。
- 无论供应商实际返回什么格式，本地最终都转换为 PNG 保存。

## 删除规则

可以删除：

- 供应商文件上传用的临时 JPG/PNG。
- 供应商原始输出临时下载文件。
- API 调用过程中生成的中间传输文件。

不能删除：

- 项目资产图片。
- Agent 对话历史附件图片。
- 已被项目资产引用的图片。

## 最终约定

- 新输入、新生成并入库的项目资产：PNG。
- 既有项目资产和历史版本：保留实际格式，不追溯转换。
- 前端查看：直接读项目原图，不依赖固定扩展名。
- Agent 对话历史附件：保留原格式，不强制转 PNG，不随意删除。
- 供应商输入：统一准备；无透明转高质量 JPG，有透明保留 PNG。
- 供应商输出：检测真实格式后统一转 PNG 保存。
- 供应商输出格式设置：只影响外部 API 返回偏好，不影响本地最终 PNG 存储。
